## 文章摘要

Usagi 是由 Brett Chalupa 开发并发布到公共领域（public domain）的一款轻量级 2D 游戏引擎，专门面向像素风格游戏的快速原型开发，核心理念是「简单」与「快速迭代」。

在技术实现上，Usagi 的底层用 Rust 编写，但对游戏开发者暴露的脚本语言是 Lua 5.5。作者解释选择 Lua 的原因是：Lua 体积小、在游戏工具链中被广泛使用，而且足够强大、不会成为开发者的阻碍。图形与输入由 Raylib 驱动，Lua 绑定则通过 mlua 实现。

Usagi 最突出的特性包括：

- **实时热重载（Live Reload）**：修改代码或资源后立即生效，且不会丢失游戏状态。开发者可以在游戏运行过程中调整精灵图或逻辑，并马上看到结果。
- **跨平台导出**：一条命令即可同时为 Linux、macOS、Windows 和 Web 打包游戏，并对运行时模板做 SHA-256 校验。
- **内置暂停菜单**：自带音量控制、全屏切换以及可自定义的按键重映射，开发者无需自己实现。
- **固定的 API 与刻意的约束**：默认分辨率 320×180、16×16 的精灵网格、单张精灵表（sprite sheet）。作者把这些限制称为「激发创意的约束」，而非局限。
- **资源管理**：项目通过分门别类的目录组织精灵、音效、音乐和数据文件，导出时无缝打包。

在使用方式上，开发者以 `main.lua` 作为入口，编写 `_init()`、`_update(dt)` 和 `_draw(dt)` 三个回调。引擎在开发期监听文件变化，保存即重新编译 Lua。状态会跨重载持续保留，除非手动重置（F5），以便在反复调参时保留游戏进度。

设计上的一些值得注意的取舍：遵循 Lua 习惯采用 1 起始的索引（`gfx.spr(1, ...)` 绘制左上角的精灵）；预处理器会把 `x += 1` 改写为 `x = x + 1`；引擎刻意不内置精灵编辑器、音效生成器或音乐编辑器，开发者自带工具；后处理着色器不会应用到截图（F8/F9），从而保留干净的像素网格截图，同时屏幕上仍可呈现 CRT 等效果。此外还提供 `usagi loveify`，可将项目一次性移植到 Love2D，以满足需要 iOS/Android 支持或更强能力的场景。

Usagi 的适用场景是：快速试验一个点子、规模超出 Pico-8 token 限制的项目、想要比 Love2D 更简单的方案，以及教学场景。它明确不适合移动端、VR 或大规模精雕细琢的成品制作。

从项目本身的定位来看，它延续了 Pico-8、TIC-80、Love2D 等「fantasy console / 极简引擎」一脉的思路：用刻意的技术约束（低分辨率、单精灵表、固定 API）来降低决策负担、加速从想法到可玩原型的过程，同时用 Rust + Lua 的组合兼顾运行时性能与脚本开发的便捷性。对于已经感到 Pico-8 token 上限掣肘、又觉得 Love2D 偏重的开发者，Usagi 试图填补中间的空档。
