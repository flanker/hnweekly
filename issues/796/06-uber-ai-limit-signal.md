## 文章摘要

Simon Willison 在本文中评论了一则新闻：Uber 在仅用四个月就耗尽其 2026 年 AI 预算后，对每个 AI 编程工具（如 Claude Code 和 Cursor）实施了每月 1500 美元的人均上限，以控制成本。

**政策内容**：每个工具有各自独立的 1500 美元预算——在一个工具上的花费不会影响另一个。该限制仅适用于 agentic（自主行动型）编程软件。

**为何重要**：Willison 认为这代表了「对超支的一种理性政策回应」，远比那种比拼 token 用量的「排行榜」式做法更可取。

**定价信号**：这个上限揭示了关于 AI 工具经济性的有意义信息。按每位工程师每月 3000 美元（两个工具）计算，年度上限约为 36000 美元——大约相当于 Uber 软件工程师中位数薪酬包（33 万美元）的 11%。

**个人参照**：Willison 指出，自己每个供应商每月的 token 消耗约 1000 美元，但因为有补贴的个人套餐，实际只花了约 100 美元——而这种安排对 Uber 这样的企业客户已不再提供。

**剩余空间**：即便按他自己的使用模式，在 Uber 的上限之下，每个工具每月仍会剩下大约 500 美元的额度。

**核心论点**：这一政策展示了各组织如何围绕 AI 工具建立理性的支出边界，同时也揭示了它们对这些工具价值的感知。

## HN 评论精华

**simonw**（Simon Willison 本人）在讨论中谈到，他最看重的资产是自己的「可信度」（credibility）：之所以有这么多人读他的文章并觉得有用，是因为他被视为一个可信的信息来源——在充斥着标题党和错误信息的世界里，他在「AI 会毁灭我们」的末日论和「AI 能解决一切」的炒作之间，占据着那块罕见的中间地带。可信度难赚易失，他已经写了 24 年博客。

**SyneRyder** 提供了一组具体数据：他用的是 100 美元的 Claude Max 套餐，用量只到套餐上限的约 50%，但过去 30 天若按 API token 价格折算，相当于花了 1850 美元。他指出，如果保存所有 Claude Code 对话，存档文件里就含有 API 成本，可以自己算——他最贵的一次会话一晚上就烧掉了超过 100 美元的 token。

**thewebguyd** 对常见的「硬件成本类比」提出修正：公司给员工花 3 到 5 千美元买硬件并不是每月成本，而是入职时一次性、之后每 3 到 5 年一次，摊销下来每月约 50 美元/人。他还担心未来订阅套餐可能出现「抽地毯」（rug pull）式涨价，无论对组织还是个人都是隐患。

**dofm** 给出一个两面下注的判断：长期看 token 价格必然下降（如果「token」仍是计价单位）；但短中期内，为了让 IPO 成功，人们必须真正为所用的东西付费，所以价格正在、也会大幅上涨，待估值确定后再从那个点缓慢回落。

**bunderbunder** 提出一个犀利观点：大部分软件开发工作其实并不盈利——项目要么被放弃、要么失败、要么上线了却没有正向 ROI。就像风险投资一样，少数成功的项目赚的钱足以覆盖其余的亏损，因此很多 AI 编程支出很可能本质上是纯粹的浪费。

**blobbers** 坦承自己确实会「token max」：写一个目标文件，然后让 claude 在循环里反复跑——既然在做研究、又有明确的 KPI，为什么不把预算花在「尝试某某技术以改进结果」上呢？这正是企业设置限额想要遏制的行为。

值得一提的是，**toraway** 在串里指认某个账号疑似 AI 机器人，并总结了识别 HN 机器人评论的若干特征，比如反复使用那句臭名昭著的机器人套话「X is real」（某某是真实存在的）等。
