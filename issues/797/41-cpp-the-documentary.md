## 文章摘要

《C++：纪录片》（*C++: The Documentary*）于 2026 年 6 月 4 日在 YouTube 首播。这部时长约 1 小时 8 分钟的影片，记录了 C++ 从诞生到今天广泛普及的四十年历程，串起了从「带类的 C」（C with Classes）到现代 C++ 的各个关键阶段。片中提到，C++ 目前是全球排名前四的编程语言中增长最快的一门，在 3.5 年内用户增长超过 90%。

影片汇集了众多重量级人物，包括 C++ 之父 Bjarne Stroustrup、标准模板库（STL）设计者 Alexander Stepanov，以及 Doom/Quake 的联合创作者 John Romero、LLVM 与 Swift 的缔造者 Chris Lattner 等多位贡献者和标准委员会成员。

纪录片以二十多个章节展开叙事，涵盖贝尔实验室的初创时期、早期采用、标准化进程（C++98）、千禧年初的「C++ 寒冬」、现代 C++ 的演进，以及面向未来的种种挑战。首播期间还设有直播聊天室，Stroustrup 等参与者亲自现身互动。

## HN 评论精华

**mirmor23**：对 C++ 的复杂性与碎片化表示怀疑，认为 Ken Thompson 当年批评 C++「不连贯、复杂、堆砌想法的垃圾堆」至今仍有共鸣。他还指出，如果没有 C 的遗产加持，C++ 不会获得如此广泛的采用，而层出不穷的《Effective C++》系列丛书恰恰印证了这门语言的晦涩难懂。

**WalterBright**：从历史角度回顾，认为 Zortech C++ 才是 C++ 在 MS-DOS 上成功的关键——它解决了 Cfront 无法处理的编译速度与指针管理问题，由此形成的临界规模促使 Borland 和 Microsoft 相继开发自己的 C++ 实现，从根本上塑造了这门语言的走向。

**wasmperson**：质疑 WalterBright 的说法，指出他本人深度参与了 Zortech 的开发，难免带有偏见，并反问若没有 Zortech 在前，MSVC++ 或 Turbo C++ 是否就不会出现。

**andrewl-hn**：提出一个有趣观点——C++ 的存在其实「拯救」了 C，因为它为那些可能撕裂 C 语言的功能请求提供了一个去处，让 C 得以维持其极简主义哲学。

**jmalicki**：盛赞 Rust 是一次更优秀的「重新发明」，兼具 C++ 的性能、函数式语言的特性以及更好的人机工程学，并强调 Rust 拒绝未定义行为这一核心理念，正是它相较 C++ 设计哲学的根本优势。
