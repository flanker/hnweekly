## 文章摘要

OpenGridWorks 是一个开源的电力基础设施可视化项目，旨在将全球的电网基础设施（包括输电线路、发电站、变电站等）绘制在交互式地图上。该项目利用 OpenStreetMap 等开放数据源，构建了一个可以浏览和探索全球电力基础设施的 Web 应用。

该平台允许用户在地图上查看各类电力设施的位置和详细信息，包括发电厂的装机容量、输电线路的路径和电压等级等关键参数。项目部署在 Vercel 平台上，采用现代 Web 技术栈构建。从 HN 讨论中可以了解到，该网站展示的数据覆盖了全球范围，但不同地区的数据密度存在显著差异——发达国家和地区的数据较为完善，而非洲等发展中地区的数据则相对稀疏。

OpenGridWorks 的核心价值在于将分散在各处的电力基础设施数据集中到一个统一的可视化平台上，使研究人员、政策制定者、能源行业从业者和公众都能方便地了解电力基础设施的分布和状态。例如，英国的装机容量数据显示为 115GW，这代表的是总装机容量而非实时输出功率。

项目的交互功能包括地图缩放、平移浏览、点击查看设施详情等。初始版本还有一个自动旋转地球仪的功能，但在收到用户反馈后已默认关闭。值得注意的是，这类似于已有的 OpenInfraMap 项目，后者同样基于 OpenStreetMap 数据映射基础设施信息。

## HN 评论精华

**技术和体验反馈：**

- **eliaspro** 指出地图缺少 OpenStreetMap 要求的正确署名标注。项目创建者 **bpbart** 确认了这个问题并表示已修复。

- **greenbit** 报告了一个 UX 问题：每次地图缩放或滑动都会在浏览器历史记录中添加一条 URL，导致浏览器历史被大量无用条目填满。

- **reconnecting** 报告了严重的 CPU 使用问题，导致系统过热。这表明地图渲染在某些设备上可能存在性能优化空间。

- **topspin** 批评了自动旋转地球仪功能"没有必要"，bpbart 回应称已根据反馈将该功能默认关闭。

**数据和内容讨论：**

- **iso1631** 对英国容量数据提出疑问，bpbart 解释 115GW 代表的是装机容量（installed capacity），而非当前输出功率。

- **boringg** 注意到非洲地区的数据覆盖非常稀疏，质疑数据可用性的地区差异问题。这反映了开放数据在全球范围内分布不均的现实。

- **nullhole** 推荐了一个类似项目 OpenInfraMap，该项目同样基于 OSM 数据映射基础设施。

**安全顾虑：**

- **ninininino** 表达了地缘政治安全方面的担忧，认为将电力基础设施如此详细地公开标注可能带来安全风险。

- **pbmonster** 和 **quickthrowman** 反驳了这一担忧，解释说输电线路等基础设施本身就是高度可见的，无法在现实中隐藏，因此地图展示不会带来额外的安全风险。

**社区协作：**

- **Johnny_Bonk** 表达了对项目合作和 GitHub 访问的兴趣。**eliaspro** 建议直接在 OpenStreetMap 上贡献数据，因为这是项目的主要数据来源。

- **mcbishop** 在悬停/点击交互中遇到困难，**pietervdvn** 推荐了 MapComplete 作为替代的基础设施查看工具。
