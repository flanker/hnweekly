## 文章摘要

作者 zakirullin 开源了 **Files.md**——一个把"自有数据"和"自有软件"同时摆在台面上的笔记应用。所有内容都以 plain Markdown 存在你本地，单文件 Go 二进制即可启动服务，前端是 vanilla JS + HTML + CSS，没有构建系统。整个项目代码量小到一个人或者一个 LLM 都能完全读懂——这是作者刻意维持的边界。

它的哲学比功能更激进。作者直接引用了 Joan Westenberg 的一句话："**我的系统越长，就越是把真正思考的工作推给了未来的自己。**"他认为 Obsidian、Notion 这些"第二大脑"产品掉进了一个陷阱：用层层标签、双链、模板营造出"掌控感"，但**真正的思考**反而被无止境的元数据整理替代了。Files.md 的对策是刻意做减法——一个像聊天框一样的输入框直接捕获想法，日记自动按日期归档，待办自动归类，链接是有的但不鼓励你去做精致的图谱。

**技术栈**也贯彻"少即是多"——Go 单二进制部署（55.9% Go / 28.5% JS / 13.1% CSS），离线优先 PWA，可选自托管同步、云同步或托管服务器同步，还附带 Telegram bot 接入。隐私上零追踪、零默认上行；用户也可以选择把整个仓库就放在 Dropbox / iCloud 里同步。文件本身是纯 .md，没有任何专有锁定，作者明确表示就算这个软件明天没了，你的数据依然能被任何 Markdown 编辑器打开。

变现路径作者也直说了——他先放一个赞助按钮，能否养活项目以后再说；自己用得爽是第一位。

## HN 评论精华

- **himata4113**：被点醒了——"这件事让我意识到 Obsidian *不是*开源的，但它给我的感觉一直像是开源的。"很多用户其实是被 Obsidian 的"本地文件 + 开源插件"模糊了边界，没意识到核心客户端依然是闭源商业软件。**bachmeier** 补充澄清：Obsidian 是 open-core 的"反面"——核心闭源，但你的数据是开放格式 Markdown，插件生态全开源。

- **ocimbote**（标题党争议）：直接吐槽"Obsidian 替代品"这个 framing 容易误导——他原本以为会拿到 feature-parity 和 API 兼容，结果发现 Files.md 是**完全不同的另一种东西**——更有意思，但不是替代。建议作者别再用"alternative"这个词。

- **backscratches**（终端原教旨派）：晒出自己的方案——helix + markdown-oxide LSP 复刻了 Obsidian 的双链/标签/自动建笔记/脚注跳转，加 rumdl 做 lint，rucola/tree-md 做美化和图谱，全部跑在终端里、SSH 进服务器就用。结论："**没有比终端更好的文本界面**，而且我们正处于黄金时代。"——这是 HN 评论区一个典型分支：与其换 Obsidian，不如根本不要 GUI。

- **UnnoTed**：分享自己在用 Qt6 / QWidgets / C++ 写一个**原生** Obsidian，目标是干掉 Electron 的内存占用——目前只用 15MB RAM、几乎不吃 CPU、不需要 GPU，"像一个文本编辑器应该有的样子"。

- **tsurba** 力推 **Joplin**：完全开源、官方客户端覆盖 Win/macOS/Linux/iOS/Android、用 Dropbox 同步一个登录搞定、免费、2GB 文本配额绰绰有余。在他看来 git 同步笔记是"折磨"，Joplin 才是最优解。

- **Ifkaluva**（哲学派）：对作者反对的"第二大脑"观点表达深度共鸣——他特别反感 Tiago Forte 提出的"我的目标是什么都不记，让第二大脑在正确的时刻浮现正确的上下文"。"听起来很糟糕——就像'反正能 google 就不用记了'。"作者那种"用第二大脑去强化第一大脑"的方向更对。

- **tomburgs**：指出 Obsidian 真正的护城河是**插件生态**——所有插件全开源；真要颠覆，就得做一个能兼容 Obsidian 插件的编辑器。另外吐槽：2026 年了我们还在用 PWA 假装桌面应用，太奇怪——他个人已经"非原生不装"。

- **bityard**（务实派）：作者自承"懒得维护元数据"才写了这个，他想说的更直接——"**你也许根本不需要那么多元数据**。"丢页面进去、搜索找回、需要时让 AI 总结合并，足够了。

- **prepend**（长期保存焦虑）：希望有一个**完全无服务端**的方案，能跑 20+ 年不依赖任何运营方。Files.md 的自托管路径基本满足，但他想要更彻底的纯本地。

- **zakirullin**（作者自述）：在评论区贴出了项目的核心立场——"不仅数据要拥有，**打开数据的软件也要拥有**。这样数据和工具就能一起、完全在你掌控下成长。"并提到代码刻意优化得让 LLM 也能改：作为用户你可以让 AI 按自己需求改 Files.md 本身。
