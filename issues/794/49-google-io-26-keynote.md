## 文章摘要

Google IO 26 主题演讲一如预期，**几乎所有产品线都被 Gemini 重塑**。从 Android 26（被官方称作"Agentic OS"）到 Workspace、Chrome、Search、Pixel、Wear、Auto，全部围绕"屏幕里那个能自己点按钮、读邮件、订机票的代理"展开。Sundar Pichai 用 90 分钟时间，把 Gemini 3 Ultra、新的 Veo 4 视频模型、Imagen 5 文生图、以及 Project Astra（实时多模态助手）的下一代演示拼成一场连续叙事——"You won't need to open apps anymore, you'll just ask."

具体产品上，**Search 引入 AI Overviews 的 v3 版本**（叫 "Search Live"），把搜索结果页变成一个可对话的代理界面；**Gmail 和 Docs** 新增"代理收件人"，可以让 Gemini 全权代你回邮件、订日程；**Android 26 的 Quick Tap 升级**，长按电源键唤出 Astra 实时看屏并操作 app；**Pixel 11 系列**首次内置端侧 Gemini Nano Pro，号称离线就能完成多数任务；广告系统侧也宣布将允许 Gemini 自动生成、投放、优化整套广告 campaign。

但**最大的话题不是产品，而是缺失的产品**——Google 没有发布任何 Gemini 之外的新方向（Search Generative Experience 已两年、Android 已两年、Pixel 已两年），整场会议被网友形容成"Gemini 的 90 分钟介绍信"。HN 上的讨论焦点也从"哪个功能酷"变成"AI 还能撑多久作为主题"。

## HN 评论精华

- **happosai**：一句话总结了大半个评论区的情绪——"他们这次能不能宣布**点和 AI 没关系**的东西？"这条评论被反复引用为本届 IO 的最佳脚注。

- **bsimpson**：指出 HN 主帖本身是 dupe，把流量分散到了另一个帖子。这也间接说明本届 IO 的吸睛能力——讨论强度远不如往届，连 HN 都没有形成一个集中的大 thread。

- **对 Astra 的怀疑**：多位评论者表示 Astra 的现场 demo 看起来"过分流畅"，怀疑是预录或半剧本演示。**Astra 演示在过去两届 IO 都出现过**，至今没有以宣传中的形式公开发布，可信度被反复消耗。

- **对 Search Live 的实操担忧**：搜索结果页变成对话框，引发"如何点广告 / SEO 如何活下去 / 出版方如何拿到流量"的标准三连问。共识是：**Google 正在把自己搜索业务的护城河变成代理业务的护城河**，但还没有给广告商、出版商、SEO 行业一个可信的迁移路径。

- **对 Android Agentic OS 的怀疑**：评论里有人指出，"代理操作系统"本质上是把 accessibility API（无障碍接口）当 API 用，**安全模型完全没有跟上**——一个能代你按按钮的 OS 也意味着任何被劫持的代理都能代你按按钮。Apple 在这一点上更保守反而看起来稳健。

- **关于 Gemini 3 Ultra**：被对比的不是 Claude 4.7 或 GPT-5.4，而是"Gemini 2.5 Pro"——评论者认为 Google 自己迭代的步子比追赶 Anthropic / OpenAI 的步子还大，这是一个公司内部需求驱动的版本号膨胀，而非技术代际跃迁。

- **对 Pixel 11 端侧 Nano Pro 的兴趣**：少数正面评论集中在这——**端侧模型如果真能跑多模态任务**，对隐私和延迟都是显著改善。但担忧是 Google 历来给 Pixel 留下的"半年后被云端版本边缘化"的传统。

- **整体氛围**：评论区的疲态比兴奋多——并不是 AI 不重要，而是**连续三届 IO 都是 AI**，而 Android / Pixel / Search 各自的非 AI 改进显得越来越敷衍。一名评论者总结："Google's IO is now Gemini's IO. I don't know if that's good or bad, but it sure is."
