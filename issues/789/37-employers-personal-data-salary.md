## 文章摘要

MarketWatch 的这篇报道揭示了一个令人不安的现象：越来越多的雇主正在利用求职者和员工的个人数据来推算他们愿意接受的最低薪资水平，从而在薪酬谈判中占据绝对优势。文章的核心关注点在于 Equifax 旗下的 "The Work Number"（工作编号）服务——这是一个大规模的就业和收入验证数据库，雇主和第三方机构可以通过它获取个人的薪资历史记录。

问题的严重性在于，许多员工根本不知道自己的薪资信息正在被共享。工资单服务商（如 ADP、Gusto 等）会自动将员工的收入数据上报给 Equifax 的 The Work Number 数据库，而雇主或其薪资服务提供商往往在员工完全不知情的情况下就已经开始共享这些数据。这些数据随后被出售给潜在雇主、房东、贷款机构等各类机构。

文章指出，当求职者走进面试室时，招聘方可能已经掌握了他们过去的确切薪资水平，这使得求职者在谈判中处于极为不利的地位。如果雇主知道你之前的收入只有某个水平，他们就可以精确地开出一个仅比你之前薪资略高的报价，而非支付该岗位的市场公允价值。

值得注意的是，员工可以选择冻结自己在 The Work Number 上的数据，但这个过程本身也颇具讽刺意味——你需要向该机构提交更多个人身份信息才能完成冻结操作。而且这只是"冻结"而非"删除"，数据仍然保留在系统中。这一现象也引发了关于数据隐私和劳动权益保护的广泛讨论，尤其是在美国和欧洲不同监管体系下的对比。欧洲的 GDPR 法规使得类似操作在欧盟境内几乎不可能合法进行，而美国的劳动者则缺乏同等程度的保护。

## HN 评论精华

1. **anonymars** 分享了一个关键的自我保护建议：可以通过 The Work Number 网站冻结自己的数据（employees.theworknumber.com/employee-data-freeze）。他指出，工资单服务本质上是把你的薪资信息"出卖"给 Equifax，也就是那个在 2017 年发生大规模数据泄露事件的同一家公司。

2. **hailvectron** 揭露了一个更加令人愤怒的细节：Gusto（一家流行的薪资服务商）在 2024 年 7 月自动启用了与 The Work Number 的数据共享功能，将其包装成"帮助员工更快获得贷款审批"的"福利"。他作为公司合伙人，发现公司团队成员在从未被征求同意的情况下，薪资信息即将被自动共享。他愤怒地选择了退出，但承认自己可能是唯一会去主动阻止此事的合伙人。

3. **xvxvx** 分享了在 Equifax 工作的亲身经历：90 年代时，有人专门去 Equifax 找那些别人都不想做的工作岗位，目的只是为了获取系统访问权限来重置自己的信用评分，而且公司一旦发现也无法回滚这些修改。

4. **delusional** 以欧洲人的视角评论道："看到一家私人公司警告说，如果你不允许他们处理你的个人数据，可能会影响你获得社会服务，这简直令人震惊。" 这引发了关于美欧数据隐私保护差异的讨论，多位评论者指出 GDPR 法规使得欧洲不可能出现类似的情况。

5. **avaer** 对"冻结而非清除"机制表示不满："我讨厌必须选择退出我从未注册、也永远不会注册的东西。" 他还指出，冻结过程本身需要上传更多个人身份信息来验证身份，这更加荒谬。
