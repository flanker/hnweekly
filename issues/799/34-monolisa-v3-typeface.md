## 文章摘要

MonoLisa 是一款专为开发者和创意工作者设计的专业字体，此次发布的 v3 版本进一步完善了它的产品体系。它由两个互补的字族组成：用于编程的等宽字体 **MonoLisa Code**，以及用于通用设计排版的比例字体 **MonoLisa Text**。整套字体强调「清晰与风格并重」，通过独特的字形和精心平衡的字间距来达成这一目标。

在设计理念上，MonoLisa 坚持「功能优先」。其核心主张是：对于一款好的编程字体而言，「字形之间的可区分性至关重要，因为它能减少歧义、帮助开发者避免犯错」。具体的设计特征包括：字形采用开放式结构，终端笔画相对舒展并指向相邻字母；整体宽度比同类等宽字体约宽 7%，从而降低视觉疲劳；并通过细致的间距补偿来平衡明暗空间的分布。字体支持超过 200 种语言，涵盖拉丁文、西里尔文、希腊文、希伯来文和亚美尼亚文等。

在技术特性方面，MonoLisa 采用可变字体技术，提供两个调节轴：「字重」轴可在单个文件内从极细（Thin）覆盖到极粗（Black）；「Grade」轴则可在不改变字形宽度的前提下微调笔画粗细。字体内置了 120 多个专门设计的连字（ligatures），并按可选功能分组——默认仅启用细微的空白连字，其余编程连字均由开发者自行「选择开启」，体现了把控制权交还给用户、避免意外的设计哲学。此外还包含 40 多项 OpenType 特性，例如上下文替换、自由连字、脚本变体，以及 Markdown 标题、箭头组合等代码专用功能。字符集还覆盖了 PowerLine 终端符号、制表符（box-drawing）、货币与数学运算符、盲文等。

MonoLisa 由 FaceType 字体工作室的字体设计师 Marcus Sterz，联合开发者 Andrey Okonetchnikov 和 Juho Vepsäläinen 共同打造。定价采用许可证模式：提供功能受限的免费试用；Developer 许可证（性价比最高）包含完整功能的 MonoLisa Code；Creator 许可证则包含 Code + Text 的完整超级字族，每个字重 49 欧元；同时还为通过在校验证的学生提供折扣。官网还提供了一个「tester」工具，让用户在下载前就能预览和自定义字体效果。

## HN 评论精华

围绕价格的争论是评论区的焦点：

- **efficax**（189 赞）质疑 149 美元的售价：在 nerdfonts.com 上已经有大量优秀的免费替代字体，凭什么值这个溢价？
- **DVT** 更直接地称这种定价是「纯粹的营销手段」，认为 Fira Code、Hack 和 JetBrains Mono 在质量上「即便不更优，也丝毫不逊色」，而且完全免费。
- **smcleod** 觉得 250 澳元的个人许可价格过高，认为 25 美元才更合理。
- **PufPufPuf** 同样发问：既然 JetBrains Mono、Iosevka 和 Fira Code 已经足够好，谁还会去花 149 美元？

但也有不少忠实用户为它辩护：

- **gregrobson** 表示自己 2022 年买下 MonoLisa 后就再也没换过字体，觉得它「非常平衡、护眼」。
- **cassidoo** 已经日常使用六年多，盛赞它较宽的字宽在教学和屏幕共享时表现最佳，直言它就是「最好的」。
- **roundcan7998** 专门注册账号表达喜爱，特别感激它为印度等地区提供了基于购买力平价（PPP）的定价。
- **piskov** 则从另一个角度论证：自己多年前买了更贵的 Operator Mono 至今天天在用——对每天都要面对的工具来说，优质付费字体物有所值。

在功能与许可方面的批评也颇有价值：

- **microflash** 因许可证禁止对字体进行子集化（subsetting）和修改而拒绝购买，认为这是无法接受的硬伤。
- **MoonWalk** 指出 Text 变体的大写「I」缺少上下横线（crossbar），不过开发者澄清可以通过样式集（stylistic set）切换替代字形。
- **warpspin** 希望能在不移除全部空白调整的前提下，单独禁用某些连字（如「<=」），并建议提供更大的点号和逗号以提升可读性。

此外，**cyphar** 对官网中关于「为 AI 时代优化」的营销文案表示怀疑，质疑用户究竟该如何判断一款字体更适合显示 LLM 生成的代码、还是传统手写代码。在对比层面，**jvican** 推荐了免费的 Input Mono，认为它「几乎在各方面都更胜一筹」。
